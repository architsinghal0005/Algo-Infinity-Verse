SharedArrayBuffer Blocked

Your browser has blocked shared memory allocation due to missing Cross-Origin Isolation headers. This is a standard security measure to prevent Spectre attacks.

How to fix for local testing:
  • Run your local server with these headers:
    • Cross-Origin-Opener-Policy: same-origin
    Cross-Origin-Embedder-Policy: require-corp
  • Or, test in Chrome by enabling this flag:
    chrome://flags/#enable-sharedarraybuffer-credentialless

Thread Controller

The Setup: Two independent Web Workers share a single block of memory (a Bank Balance). Thread 1 deposits money, Thread 2 withdraws money simultaneously.

1. Race Condition (No Locks)

Both threads read and write memory blindly. Context switching causes them to overwrite each other's data, corrupting the balance.

2. Thread-Safe (Mutex Locks)

Threads use Atomics.wait() to acquire a Mutex lock before touching the memory, ensuring operations happen sequentially.

3. Mutual Deadlock

Thread 1 locks Mutex A and waits for B. Thread 2 locks Mutex B and waits for A. Both freeze infinitely.
